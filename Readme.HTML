نظام إدارة الطعون والجلسات

هذه الصفحة تقدم وصفًا مختصرًا لوظائف التطبيق وطريقة استخدامه على مستوى التشغيل اليومي والإعدادات العامة.

إدارة الدعاوى والطعون أجندة الجلسات ترحيل الجلسات الأحكام والمرفقات رفع سحابي

نظرة عامة

التطبيق مخصص لإدارة الدعاوى والطعون والجلسات والأحكام والمذكرات والمرفقات والتقارير داخل واجهة واحدة. يركز النظام على تسهيل المتابعة اليومية، خاصة في ملفات القضايا التي تتكرر فيها الجلسات والقرارات والترحيل.

أهم الوظائف

  • تسجيل الدعاوى والطعون وبيانات الخصوم والرول والسنة والموضوع والحالة.
  • عرض أجندة الجلسات مع فلاتر وترتيب وتحديد متعدد وترحيل فردي أو جماعي.
  • متابعة آخر جلسة والجلسة الحالية وسجل القرارات السابقة.
  • إدارة الأحكام والقرارات وربطها بملف الدعوى.
  • رفع المرفقات وملفات الرولات والمنشورات إلى Google Drive أو OneDrive وفق إعدادات النظام.
  • استيراد البيانات من ملفات Excel أو CSV مع مزامنة وتقليل التكرار.
  • عرض تقارير وإحصائيات تشغيلية عن الحالات والجلسات والأحكام.
  • إدارة القوائم المرجعية والهوية العامة للتطبيق وإعدادات التخزين السحابي.

الفرق بين أجندة الجلسات والسجل اليدوي

  • أجندة الجلسات: هي شاشة العمل اليومية الأساسية، وتشمل الفلاتر والترحيل والمتابعة السريعة وآخر جلسة.
  • السجل اليدوي للجلسات: سجل فرعي يستخدم عند الحاجة لإضافة جلسة يدويًا أو أرشفة جلسات منفصلة.

التخزين السحابي

يدعم النظام رفع الملفات إلى Google Drive وOneDrive. في الوضع المعتاد، يقوم كل مستخدم بربط حسابه الرسمي ثم ترفع الملفات إلى مساحته السحابية الخاصة به، بينما يحتفظ النظام داخل قاعدة البيانات ببيانات الملف والرابط المرتبط بالدعوى أو الجلسة أو الحكم.

متطلبات التشغيل

  • متصفح حديث يدعم JavaScript.
  • اتصال صالح بخدمات Firebase المستخدمة في المشروع.
  • إعداد صالح للربط السحابي إذا كان رفع الملفات إلى Google Drive أو OneDrive مطلوبًا.
  • صلاحيات مناسبة للمستخدم داخل النظام وفق آلية الإدارة المعتمدة.

ملاحظات مهمة

  • هذا المستند وصفي عام، ولا يغني عن سياسة الخصوصية أو الشروط.
  • أي استخدام للبيانات أو رفع للمرفقات يجب أن يراعي سياسات الجهة المالكة للنظام.
  • ينصح بمراجعة ملفات PRIVACY.HTML وterms.html.
